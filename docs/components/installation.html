Installation

Add hallucn components to your project using the shadcn CLI.

Prerequisites

hallucn components require Tailwind CSS v4 in your project. The components emit Tailwind utility classes and rely on your build system to process them.

  • Tailwind CSS v4 configured in your project
  • A module bundler (Vite, webpack, esbuild, etc.)
  • components.json configured (for shadcn CLI)

Install Components

Use the shadcn CLI to add components directly from the hallucn registry. Each component is installed as a source file you can customize. 

npx shadcn@latest add https://hallucn-ui.github.io/hallucn-ui/r/button.json

This copies the component source to src/components/ui/button.ts and installs required dependencies.

Installing Multiple Components

You can install multiple components in a single command: 

npx shadcn@latest add \
  https://hallucn-ui.github.io/hallucn-ui/r/button.json \
  https://hallucn-ui.github.io/hallucn-ui/r/dialog.json \
  https://hallucn-ui.github.io/hallucn-ui/r/input.json

Configure Registry Namespace

For a shorter command, configure hallucn as a named registry in your components.json: 

{
  "registries": {
    "@hallucn": "https://hallucn-ui.github.io/hallucn-ui/r/{name}.json"
  }
}

Then install components with shorter commands: 

npx shadcn@latest add @hallucn/button @hallucn/dialog

What Gets Installed

When you install a component, the CLI adds:

Item Location Component source src/components/ui/{component}.ts or .js Utils (first install) src/lib/utils.ts or .js Dependencies package.json (lit, etc.)

File extension (.ts or .js) is determined by your components.json tsx setting.

Usage

Import the component in your app's entry point to register the custom element: 

// main.ts or app entry point
import "./components/ui/button"

Then use it anywhere in your HTML: 

<hal-button variant="outline">Click me</hal-button>

Dependencies

Components automatically install their required npm packages:

Package Used By lit All components (web component framework) clsx, tailwind-merge All components (class utilities) class-variance-authority Components with variants (button, badge, etc.) @floating-ui/dom Positioned components (tooltip, popover, select, etc.)

TypeScript Configuration

If using TypeScript ("tsx": true in components.json), ensure your tsconfig.json includes decorator support: 

{
  "compilerOptions": {
    "experimentalDecorators": true,
    "useDefineForClassFields": false
  }
}

JavaScript Configuration

To use JavaScript instead of TypeScript, set "tsx": false in your components.json: 

{
  "tsx": false
}

The shadcn CLI automatically transforms TypeScript to JavaScript when installing components. Decorators are preserved and type annotations are stripped.

For import aliases, configure a jsconfig.json: 

{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@/*": ["./src/*"]
    }
  }
}
